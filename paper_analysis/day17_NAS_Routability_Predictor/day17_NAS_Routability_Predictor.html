1. 背景与动机

1.1 ML for EDA 的"人海战术"困境

机器学习在 EDA 领域的广泛应用带来了显著的效率提升，但模型开发本身面临严重瓶颈：

困境说明
专业知识双重门槛开发者需要同时精通 ML（架构设计、超参调优）和 EDA（特征工程、领域知识）
开发周期漫长设计一个工业级 CNN 预测器，经验丰富的开发者也需数周到数月
架构探索受限人类设计师倾向于使用层次化结构，极少尝试并行分支或异构操作组合
可迁移性差针对特定任务调优的架构难以直接迁移到新场景

1.2 现有可布线性预测器的共同局限

局限说明示例
操作类型单一仅使用常规卷积，缺少大感受野操作RouteNet 仅用 3×3 卷积
层次化结构受限编码器-解码器框架，分支结构极少U-Net 仅有一条跳跃连接
无法捕获跨尺度模式拥塞由远近不同区域共同影响，单一尺度难以覆盖大范围绕线模式被忽略
核心洞察：拥塞预测的正确架构可能远比人类直觉设计的更复杂。我们需要一种系统化的方法来探索庞大的架构空间——这就是 NAS 的价值所在。

4. 问题建模

4.1 输入表示

布局放置完成后，将布局划分为 $w \times h$ 的网格，每个 tile 提取 $c$ 个特征通道：

$$X_i \in \mathbb{R}^{w \times h \times c}, \quad c = 17$$

4.2 问题 1：违反网数预测

$$f_{A_C}: X_i \in \mathbb{R}^{w \times h \times c} \rightarrow y_i \in \mathbb{R}$$

评估指标：Kendall's $\tau \in [-1, 1]$，捕获预测排序与真实排序的相关性。

4.3 问题 2：DRC 热点检测

$$f_{A_L}: X_i \in \mathbb{R}^{w \times h \times c} \rightarrow Y_i \in \{0, 1\}^{w \times h}$$

评估指标：ROC-AUC（ROC 曲线下面积），衡量分类器的诊断能力。

两个问题的本质区别：违反网数预测是全局回归（一张布局→一个标量），DRC 热点检测是像素级分割（一张布局→一个二值掩码）。NAS 为两个任务分别搜索不同的最优架构。

5.2 搜索空间

5.2.1 候选操作

操作符号参数特点
常规卷积 (32 filters)C323×3, 32 filters基础特征提取
常规卷积 (64 filters)C643×3, 64 filters更丰富的特征表示

5.2.2 Guide-DAG 和 Sampled-DAG

Guide-DAG $G_i(V_i, E_i)$：

  • 每个顶点 $v \in V_i$ 代表一个候选操作 $\text{OP}_v$
  • 顶点完全有序：$1 \to 2 \to \cdots \to 7$
  • 所有满足 $u < v$ 的边都存在 → 全连接 DAG

顶点计算规则：

$$o_v = \text{OP}_v(\text{Concat}(i_{u_1}, i_{u_2}, \ldots, i_{u_k}))$$

5.3 评估策略

采用 Proxyless 评估：直接在目标数据集上搜索。

任务搜索目标
违反网数预测验证集上的 Kendall's $\tau$
DRC 热点检测验证集上的 ROC-AUC

5.4 搜索策略

5.4.1 边权更新公式

$$w_e = w_e \cdot \exp(\alpha(\mu - \beta))$$

其中 $w_e$ 为边权，$\alpha$ 为学习率，$\mu$ 为当前评估指标，$\beta$ 为基线指标（历史 top-5 平均）。

直觉：若当前模型优于基线（$\mu > \beta$），被选中边的权重增加；指数函数放大更新幅度，加速收敛。

5.4.2 Algorithm 1：连接权重更新

输入: Guide-DAGs {G_i}, 基线指标 β, 学习率 α
1:  for i = 1 to 6 do
2:    G'_i = Preprocess(G_i)  // 插入伪顶点 v_0
3:  while not converge do
4:    for i = 1 to 6 do
5:      S_i = Sampling(G'_i)  // Algorithm 2
6:    M = ConstructModel({S_i})
7:    μ = Eval(M)
8:    for i = 1 to 6 do
9:      for all edge e in G'_i do
10:       if e is selected in S_i then
11:         w_e = w_e · exp(α(μ - β))
12: β = average of top five metrics

5.4.3 Algorithm 2：子图采样

输入: G'_i(V'_i, E'_i)
输出: S_i(V_{S_i}, E_{S_i})
1:  V_{S_i} = {v_0}  // 从伪顶点开始
2:  E_{S_i} = ∅
3:  for each v_j ∈ V'_i do
4:    if v_j ∈ V_{S_i} then
5:      for each e(v_j, v_k) ∈ E'_i do
6:        p = exp(w_e(v_j,v_k)) / Σ_l exp(w_e(v_j,v_l))  // softmax
7:        r ~ Uniform(0, 1)
8:        if p > r then  // 按概率采样
9:          V_{S_i} = V_{S_i} ∪ {v_k}
10:         E_{S_i} = E_{S_i} ∪ {e(v_j, v_k)}
11: return S_i

6. 特征提取

6.1 特征总览

输入特征张量 $X \in \mathbb{R}^{w \times h \times 17}$：

类别通道数特征列表
单元密度5宏单元密度、全部单元密度、DFF 密度、时钟树单元密度、引脚密度
线密度126 种 × 2 组（大扇出/小扇出）

6.2 线密度特征详解

网按扇出大小分为两组，每组提取 6 种特征：

飞行线类型连接方式捕获的信息
成对飞行线每个引脚到同网所有其他引脚全连接拓扑
星形飞行线每个引脚到引脚中心网的几何中心关系
源-汇飞行线源引脚到所有汇引脚时序优化驱动的布线方向
MST 飞行线最小生成树的所有边最短布线长度估计
设计哲学：前三类飞行线倾向于高估布线用量，MST 飞行线提供更精确的下界估计。四种飞行线互补，为模型提供不同角度的布线需求信息。

8. NAS 生成架构深度分析

8.1 违反网数预测架构

位置操作组合分析
S1, S2C64 作为父顶点宽卷积层提供丰富初始表示
S3, S42 空洞卷积 + 1 混合卷积，紧凑全局回归只需提取大范围模式
S5, S6常规卷积 + shortcut简单聚合传递到回归头
核心发现：对于全局回归任务，NAS 偏好前宽后窄的结构——早期用大通道数捕获丰富特征，中后期用大感受野操作提取全局模式，最后简单聚合。

8.2 DRC 热点检测架构

位置操作组合分析
S1, S2空洞+混合+常规卷积，复杂交互需同时捕获多尺度局部+全局模式
S3, S4每条分支至少一个空洞卷积大感受野对热点定位至关重要
S5, S6大量混合卷积层组合多尺度特征对像素级预测至关重要
核心发现：DRC 热点检测的架构远比违反网数预测复杂，特别是靠近分割头的部分。这反映了像素级预测需要同时利用多尺度模式的本质特征。

10. 创新点深度分析

10.1 设计哲学：从"设计模型"到"设计搜索空间"

传统范式：

领域知识 → 人工设计模型架构 → 训练 → 预测器

本文范式：

领域知识 → 设计搜索空间 → NAS 自动搜索最优架构 → 训练 → 预测器
关键转变：人类的角色从"直接设计架构"变为"设计搜索空间"。后者需要更高级的抽象能力——不是决定"用什么"，而是决定"从哪些中选择"。

10.2 DAG 搜索空间的巧妙设计

设计选择理由
全连接 DAG提供最大拓扑灵活性
7 个顶点平衡搜索空间大小和表达能力
6 个并行 sampled-DAG 对在不同尺度上独立搜索
拼接而非加法融合保留不同路径的完整信息
伪顶点连接下采样层允许跳过某些操作直接传递
