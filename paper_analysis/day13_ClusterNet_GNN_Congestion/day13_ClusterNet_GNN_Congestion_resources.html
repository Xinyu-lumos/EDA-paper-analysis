3. 快速上手指南（复现路线）

3.1 环境搭建

# 创建 conda 环境
conda create -n clusternet python=3.9
conda activate clusternet

# 安装 PyTorch (CUDA 版本)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu118

# 安装 PyTorch Geometric
pip install torch-scatter torch-sparse torch-cluster torch-spline-conv -f https://data.pyg.org/whl/torch-2.0.0+cu118.html
pip install torch-geometric

# 安装 Leiden 算法
pip install leidenalg igraph

# 安装 OpenROAD (需要编译或使用 Docker)
# 参考: https://github.com/The-OpenROAD-Project/OpenROAD

3.2 数据准备

# 1. 使用 OpenROAD 运行布局+布线，生成训练数据
# 输入: DEF/LEF/Verilog 文件
# 输出: 拥塞图 (congestion map) + 布局坐标

# 2. 从 DEF 文件提取网表图
python extract_netlist_graph.py --def_path design.def --lef_path tech.lef

# 3. 运行 Leiden 聚类
python run_leiden.py --graph netlist_graph.pkl --resolution 1.0

# 4. 生成簇级特征
python gen_cluster_features.py --graph netlist_graph.pkl --clusters clusters.pkl --placement placement.pkl

3.3 模型训练

# 训练 ClusterNet 模型
python train.py \
    --data_dir ./data/ \
    --epochs 200 \
    --batch_size 32 \
    --lr 0.001 \
    --gat_layers 2 \
    --gat_heads 4 \
    --hidden_dim 128

3.4 推理与优化

# 预测拥塞
python predict.py --model best_model.pth --design new_design/

# 聚类填充优化
python cluster_padding.py \
    --congestion predicted_congestion.pkl \
    --clusters clusters.pkl \
    --threshold 0.8 \
    --lambda 0.5

# 重新运行详细布局
openroad -python detailed_placement.py --design optimized_design/

4. 相关开源项目

leidenalg — Leiden 算法 Python 实现

仓库: github.com/vtraag/leidenalg

作者: V.A. Traag (Leiden 算法论文作者)

语言: Python / C++

星标: ~800+

许可证: BSD-3-Clause

说明: ClusterNet 核心依赖。实现了 Leiden 社区检测算法，支持 modularity / CPM 等多种质量函数。

PyTorch Geometric (PyG)

仓库: github.com/pyg-team/pytorch_geometric

语言: Python

星标: ~20k+

许可证: MIT

说明: 图神经网络框架，提供 GAT / GCN / GraphSAGE 等层的实现。ClusterNet 的 GAT 部分可直接使用 GATConv。

OpenROAD — 开源物理设计平台

仓库: github.com/The-OpenROAD-Project/OpenROAD

语言: C++ / Python / Tcl

星标: ~1.5k+

许可证: BSD-3-Clause

说明: 提供完整的物理设计流程（布局、布线、时序分析）。用于生成训练数据和执行优化后的详细布局。

CongestionNet — 前驱工作

论文: VLSI-SoC 2019

说明: 首次将深度 GNN 用于拥塞预测。ClusterNet 的直接前驱，但代码未公开。

DREAMPlace — GPU 加速布局器

仓库: github.com/limbo018/DREAMPlace

说明: Day 1 分析的论文对应代码。可作为布局引擎替代 OpenROAD，GPU 加速的解析布局器。
