1.2 快速上手

# 1. 克隆仓库（含子模块）
git clone --recursive https://github.com/NVlabs/AutoDMP.git
cd AutoDMP

# 2. 使用 Docker 构建（推荐）
docker build -t autodmp .
docker run --gpus all -it autodmp bash

# 3. 运行多目标贝叶斯优化（NVDLA benchmark，20 轮，2 并行 worker）
./tuner/run_tuner.sh 1 1 test/nvdla_nangate45_51/configspace.json \
  test/nvdla_nangate45_51/NV_NVDLA_partition_c.aux \
  test/nvdla_nangate45_51/nvdla_ppa.json "" 20 2 0 0 10 ./tuner \
  test/nvdla_nangate45_51/mobohb_log
提示：修改 tuner/configspace.json 可以自定义搜索空间——这是探索不同参数组合的最简单方式。

5. 关键代码解读：贝叶斯优化调参流程

5.1 搜索空间定义（configspace.json）

configspace.json 定义了 AutoDMP 优化的超参数及其搜索范围：

{
  "density_weight": {"type": "float", "min": 1e-5, "max": 1e-1, "log": true},
  "density_weight_decay": {"type": "float", "min": 1.01, "max": 1.5},
  "macro_weight": {"type": "float", "min": 1e-2, "max": 1e2, "log": true},
  "initial_learning_rate": {"type": "float", "min": 0.1, "max": 10.0},
  "macro_repulsion_radius": {"type": "float", "min": 0.5, "max": 5.0}
}
修改建议：如果你的设计有特殊特征（如大量小宏或少量超大宏），可以缩小搜索范围以加速收敛。

5.2 调参主循环（tuner_train.py 核心逻辑）

# 伪代码：AutoDMP 调参主循环
for iteration in range(max_iterations):
    # 1. 采集函数建议下一组参数
    config = bo.suggest()

    # 2. 运行 DREAMPlace 布局
    result = run_dreamplace(config, benchmark)

    # 3. 评估布局质量（多目标）
    hpwl = result["hpwl"]
    congestion = result["congestion"]
    density_overflow = result["density_overflow"]

    # 4. 更新贝叶斯优化模型
    bo.observe(config, (hpwl, congestion, density_overflow))

    # 5. 记录最优结果
    if is_pareto_optimal(config):
        save_best_config(config)
